# ⚡ Strategi Ujian OSN-K Informatika

Kompetisi olimpiade bukan hanya tentang siapa yang paling pintar ngoding, tapi siapa yang **punya strategi manajemen 150 menit paling efektif!** Mengerjakan rentetan baris kode hanya dengan pensil membutuhkan trik dan kebiasaan mental. Di folder ini, kamu akan mempelajari rahasia mempercepat waktu hitungmu di soal OSN-K.

Optimalkan waktumu untuk mendapatkan skor maksimal dari 3 part yang dikompetisikan.

## 🧭 Panduan Utama

- [⏰ Manajemen Waktu 150 Menit](./manajemen-waktu-150menit.md)  
  *Harus mulai dari ngerjain yang mana? Mending Part A, Part B, atau nyerobot kode Part C dulu? Temukan alokasi waktunya di sini.*

- [🧩 Teknik Mengerjakan Part A (Bebras Type)](./teknik-mengerjakan-part-a.md)  *(Draft)*  
  *Cara membaca teks soal panjang tanpa pusing, jurus eliminasi opsi ganda, dan menggambar grafik relasi.*

- [📊 Teknik Mengerjakan Part B (Case Analysis)](./teknik-mengerjakan-part-b.md) *(Draft)*  
  *Format membuat tabel trace waktu dan memahami state transisi pada soal simulasi kasir / antrian.*

- [💻 Teknik Mengerjakan Part C (Code Tracing)](./teknik-mengerjakan-part-c.md) *(Draft)*  
  *Menggambar pohon Call Stack rekursi, mendaftar tabel memori variabel, dan "membaca pikiran si pembuat soal".*

- [🚨 Common Mistakes (Kesalahan Umum)](./common-mistakes.md) *(Draft)*  
  *Lupa reset indeks, salah menghitung batas loop (off-by-one error), atau terkecoh `i++` vs `++i`.*
